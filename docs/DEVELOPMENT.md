## 经验教训

1. **配置源必须单一** — 多源导致漂移
2. **内部消息必须绕过** — 系统消息污染路由统计
3. **灰度发布必须渐进** — 10%→50%→100% 带观察窗口
4. **后置检查是必需的** — 应用策略需要验证
5. **审计跟踪是不可协商的** — 每次配置变更必须记录
